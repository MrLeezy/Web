# 终极映射表规则

本文档基于 [`/Users/lizhongyu/程序/团队协作系统/data/opportunity-master/映射表.xlsx`](/Users/lizhongyu/程序/团队协作系统/data/opportunity-master/映射表.xlsx) 整理，用于长期维护“商机终极表”的字段口径、来源表定位和特殊处理规则。

## 输出规则

- 每次处理后的数据必须合并在同一个结果表里。
- 不按来源拆分多个结果 sheet。
- 最终工作簿只保留一个主结果 sheet，用于承载全部来源合并后的数据。
- 如需保留处理统计信息，只能以内嵌说明形式附着在主结果表中，不能再额外拆出 summary sheet。
- 当同一批次提供了多个来源文件时，所有来源文件都要一起进入同一个终极结果表，不能只保留最后一次新增的文件。

## Campaign 衍生规则

- `Campaign Name` 和 `Allocaida ID` 不再使用原始来源表中的值。
- 这两个字段统一在 `channel` 映射完成后，按最终 `channel` 二次派生。
- 当前长期规则如下：
  - 如果 `channel` 属于 `douyin`、`toutiao`、`kuaishou`、`xiaohongshu`、`xiaohongshu_xcx`
    - `Campaign Name = FY27Q2_APJC_GC_CN Eshop_Promotion DY/Toutiao/Kuaishou`
    - `Allocaida ID = 3445890`
  - 如果 `channel` 属于 `wechat-dsp`、`wechat-dsp-xcx`
    - `Campaign Name = FY27Q2_APJC_GC_CN Eshop_Promotion Tencent-WPP`
    - `Allocaida ID = 3445892`
  - 其他所有 `channel`
    - `Campaign Name = FY27Q2_APJC_GC_CN Commerical Membership MA`
    - `Allocaida ID = 3445891`
- 该规则默认长期保留，后续如果业务变更，再统一调整此处口径和对应脚本实现。

## 特殊处理

### FY27Q1 网站数据过滤

- 在 `FY27Q1数据明细-唯一` 中，如果以下任一字段包含 `商城订单`，该行不进入最终表：
  - `LV2 来源模块`
  - `栏目模块`
  - `栏目模块 `
        - `来源表单`

### FY26Q4 周报唯一数据读取

- 当输入文件为 `FY26Q4网站及注册来源数据 follow up数据来源2026.1.30.xlsx` 时，网站来源固定读取 `FY26Q4数据明细-周报用唯一`。
- 该 sheet 按网站类来源处理，业务映射口径与 FY27Q1 网站来源一致。

### FY26Q3 双 sheet 合并读取

- 当输入文件为 `FY26Q3网站及注册来源数据 follow up数据来源2025.10.31-to carl.xlsx` 时，网站来源同时读取：
  - `数据明细`
  - `腾讯数据`
- 两个 sheet 按相同字段口径标准化后纵向合并。
- 合并后统一进入同一个终极结果表，不再拆分。

### FY26Q2 Sino 外呼读取

- 当输入文件为 `FY26Q2网站及注册来源数据Sino followup-2025.8.1.xlsx` 时，网站来源固定读取 `Sino外呼数据明细`。
- 该 sheet 按网站类来源处理，业务映射口径与其他网站来源一致。

### 客服表读取兼容

- `finished-solution` 两个文件可能存在 workbook `dimension` 异常。
- 默认先用 `openpyxl` 的只读模式读取。
- 如果只读模式误判为 1 行空表，则回退到普通模式重新读取。
- `chat` 类型文件兼容两种表头模式：
  - `A1` 开始
  - `B2` 开始
- 程序会自动识别表头位置后再取数。

### 历史 chat / 400 合并

- 历史 `chat` 文件与历史 `400` 文件都会一起进入同一个终极结果表。
- 不区分季度拆表，不区分文件拆表。
- 只要文件被纳入当前批次输入，就统一纵向合并。

### Chat + 400 汇总表读取

- 对于 `FY27Q2-400&chat数据汇总*.xlsx` 这类单表汇总文件，固定读取 `Sheet1`。
- 通过 `数据来源` 字段行内分流：
  - `400电话`、`400电话-CMG` -> `商机渠道 = 400`
  - `sales_chat` -> `商机渠道 = Chat`
- 该来源字段映射规则：
  - `manster_user_id`、`member_id`、`title`、`department`、`street Address` 固定留空
  - `email = Email`
  - `mobile Phone = Mobile Phone`
  - `account id = Dell Salesforce Account ID`
  - `company = Company`
  - `city = City`
  - `province = State`
  - `leads id = MKT ID`
  - `utm_campaign = 广告名称 (utm_campaign)`
  - `utm_content = 广告内容 (utm_content)`
  - `utm_term = 广告关键字 (utm_term)`
  - `utm_medium = 广告媒介 (utm_medium)`
  - `utm_source = 广告来源 (utm_source)`
  - `capture date = Call Date`
  - `call date = Call Date`
  - `call wk = Call Wk`
  - `call quarter = Call Quarter`
  - `call type = Call Type`
  - `remark type = Remark Type`
- 该来源仍然自动计算 `channel_classification` / `channel`。
- `Campaign Name` 与 `Allocaida ID` 对该来源固定写死为：
  - `Campaign Name = FY27Q2_APJC_GC_CN Commerical Membership MA`
  - `Allocaida ID = 3445891`

### 订单 utm 拆解

- 从 `utm参数` 中解析：
  - `utm_campaign`
  - `utm_content`
  - `utm_term`
  - `utm_medium`
  - `utm_source`
- 支持完整 URL query string 和裸参数串两种输入形式。
